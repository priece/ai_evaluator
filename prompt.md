# 需求设计
## AI评委服务
- nextjs框架开发
- 提供api接口
- 提供前端页面
- windows系统部署
- 接口文档

## 功能集合
- 获取本地摄像头信息，可选择摄像头
- 本地摄像头数据采集，前端页面实时显示
- 前端显示摄像头内容，用websocket实时传输
- 需要记录每个评估的场次轮次评估值
- 本地数据库，sqlite3，数据表有
  - 常规评估列表：
    - 主键ID：uuid
    - 场编号：uuid
    - 轮次：序号，从0开始
    - 评估结果：浮点数，0-100之间
    - 评估时间：年月日时分秒
  - 专家评估列表：
    - 主键ID：uuid
    - 场编号：uuid
    - 轮次：序号，从0开始
    - 专家评估结果：浮点数，0-100之间
    - 评估时间：年月日时分秒
  - 场次表：
    - 主键ID：uuid
    - 场编号：uuid
    - 场名：字符串

## 接口
- 摄像头接口：
  - 获取本地摄像头信息接口：
    - 请求参数：无
    - 返回参数：成功/失败，摄像头列表
- 视频采集接口：
  - 开始采集接口：
    - 请求参数：摄像头ID
    - 返回参数：成功/失败
  - 停止采集接口：
    - 请求参数：无
    - 返回参数：成功/失败
  - websocket接口：
    - 视频数据传输接口：
      - 客户端请求：发送视频数据
      - 服务器响应：返回处理结果
- 常规评估接口
  - 提交常规评估接口：
    - 请求参数：场编号、轮次
    - 返回参数：成功/失败，本次评估结果
  - 获取所有常规评估结果接口：
    - 返回：成功/失败，评估结果列表
- 专家评估接口
  - 提交专家评估接口：
    - 请求参数：场编号、轮次、专家评估值
    - 返回参数：成功/失败，本次评估结果
  - 获取所有专家评估结果接口：
    - 返回：成功/失败，专家评估结果列表
  
## 前端页面布局
- 主页面
  - 左侧菜单栏
    - 普通评估
    - 专家评估
    - 评估记录
  - 右侧内容区：
    - 菜单栏对应的页面内容区
- 普通评估页面：
  - 标题：普通评估
  - 出现在内容区
  - 左侧：摄像头数据显示区域
    - 摄像头选择下拉框
    - 开始采集按钮
    - 停止采集按钮
    - 视频显示区域
  - 右侧：评估按钮、评估结果显示区域
    - 新建场次：点击按钮【新建场次】，新建一个评估场次，生成一个新的场编号
    - 从菜单进入评估页面，默认是新建场次
    - 加载场次：点击按钮【加载场次】，弹窗列表选择历史场次
    - 评估按钮：点击触发常规评估
    - 场编号：自动生成的uuid
    - 场名：可以修改场名
    - 评估结果显示区域：显示本次评估结果
    - 本场之前的评估结果列表
- 专家评估页面
  - 标题：专家评估
  - 布局和主页面相同
  - 独立路由：/expert
  - 右侧评估区
    - 增加一个参考分数输入框，用于输入专家评估的参考分数
    - 评估按钮：点击触发专家评估
- 提交评估：点击按钮【开始评估】，开始评估，评估完成后显示分数
- 本次评估完成后，刷新评估列表
- 评估记录
  - 出现在内容区
  - 列表结构
    - 字段：场编号、轮次、评估结果、评估时间
